# docs運用ルール

更新日: 2026-04-04

## 基本方針

- 要件整理や仕様変更は、同じテーマの文書ならできるだけ1ファイルに集約する
- 類似内容で `*_re1.md` `*_v2.md` のような派生ファイルは、原則として増やさない
- 改訂が入った場合は、元ファイルの末尾に `rev1` `rev2` のように追記して履歴を残す
- 改訂前に、backend-api-function-map.mdおよびfrontend-mainjs-function-map.mdを参照して、重複がないようにする。

## 要件docの扱い

- まず元の要件docを正本として扱う
- 修正点だけ別ファイルに逃がさず、正本に追記する
- 変更内容は「何が変わったか」が分かるように、変更点と最新要件をセットで書く
- 操作フローが変わる場合は、最新フローも同じファイル内に追記する

## 今回の具体ルール

- `docs/photo-upload-requirements.md` を正本とする
- 以後の改訂は、同ファイルに `rev1` `rev2` のように追記する
- 別の revision ファイルを作った場合は、正本へ統合したあと削除する

## 命名ルール

- テーマ名はできるだけ短く明確にする
- docs は `kebab-case` のファイル名にする
- rule 配下のルール文書も `kebab-case` にする
