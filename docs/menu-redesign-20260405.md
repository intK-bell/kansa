# メニュー再設計メモ

対象: `frontend/index.html`, `frontend/main.js`, `frontend/styles.css`  
更新日: 2026-04-29  
目的: グローバルメニューの混雑を解消し、日常操作を `app-header` に寄せる。

## 1. この変更でやること

- 右上メニューをフラット構成から階層構成へ変更する。
- 最初は1階層目だけ表示し、親項目押下でアコーディオンのように子項目を開く。
- `app-header` の固定表示を見直し、状態表示を兼ねた切替UIへ変更する。
- `folder-create-card` は本文常設をやめ、`modal-card` で開く。
- `folder-list-card` は本文から外し、フォルダ操作は `app-header` とメニュー内モーダルへ寄せる。

## 2. app-header の方針

`app-header` には次を表示する。

- `current-name` は残す
- `current-room` の固定テキスト表示は削除する
- `現在のお部屋` のプルダウンを追加する
- `現在のフォルダ` のプルダウンを追加する

### 2-1. 現在のお部屋

- ラベル: `現在のお部屋`
- UI: `select`
- 選択候補: 自分が所属中のお部屋一覧
- 選択変更時: 即切替はせず、確認ダイアログを出す

確認文言:

`現在のお部屋を切り替えます。よろしいですか？`

- `OK`: 部屋切替を実行
- `キャンセル`: 変更前の選択へ戻す

### 2-2. 現在のフォルダ

- ラベル: `現在のフォルダ`
- UI: `select`
- 選択候補: 現在のお部屋内のフォルダ一覧
- 選択変更時: 既存どおり対象フォルダを切り替える
- フォルダに鍵がある場合や新着表示の扱いは既存仕様を維持する

## 3. グローバルメニュー構成

初期表示は1階層目のみとする。

- `お部屋`
- `フォルダ`
- `アカウント`
- `ログアウト`
- `使い方・料金`

親項目押下で、その配下のみ表示する。
同時に複数開くか、常に1つだけ開くかは実装時にUIを見て調整するが、少なくともフラット一覧には戻さない。

### 3-1. お部屋

- `管理`
- `作成`
- `メンバーをやめる`

対応:

- `管理`: 既存の `お部屋管理` を開く
- `作成`: 既存の `room-create-modal` を開く
- `メンバーをやめる`: 既存の leave 処理を使う

`お部屋切替` はメニューから外し、`app-header` 側に移す。

### 3-2. フォルダ

- `作成`
- `パスワード`
- `PPT出力`

対応:

- `作成`: 新しい `folder-create-modal` を開く
- `パスワード`: 新しい `folder-password-modal` を開く
- `PPT出力`: 現在選択中フォルダを対象に出力する

確認文言:

`現在のフォルダをPPTに出力します。よろしいですか？`

`フォルダ切替` はメニューから外し、`app-header` 側に移す。

### 3-3. アカウント

- `テーマ`
- `ユーザー名変更`
- `アカウント削除`

`設定` グループは作らず、テーマは `アカウント` 配下に含める。

## 4. モーダル方針

### 4-1. フォルダ作成モーダル

- 表示形式: `modal-card`
- 内容: 既存 `folder-create-card` の内容をそのまま流用
- 含む要素:
  - フォルダ名入力
  - フォルダパスワード入力
  - 作成ボタン
  - 閉じるボタン

### 4-2. テーマモーダル

- 表示形式: `modal-card`
- 起点: `アカウント > テーマ`
- 含む要素:
  - `季節` のプルダウン
  - `ダークモード` のプルダウン

`ダークモード切替` の単独メニューは廃止する。

想定選択肢:

- 季節: `春` / `夏` / `秋` / `冬`
- ダークモード: `OFF` / `ON`

反映は既存の `localStorage` とテーマ適用ロジックを流用する。

## 5. 本文エリアの扱い

### 5-1. folder-create-card

- 本文から外す
- メニューの `フォルダ > 作成` からモーダル表示に置き換える

### 5-2. folder-list-card

- 本文から外す
- フォルダ切替は `app-header` の `現在のフォルダ` に一本化する
- パスワード設定は `フォルダ > パスワード` のモーダルへ移す

### 5-3. folder-admin-list

- `お部屋管理` 内のフォルダ一覧は、引き続き管理者向けUIとする
- 各行の操作は管理者のときだけ表示する
- フォルダとメンバーが増える前提で、全フォルダのカードを一括表示しない
- 先頭に `フォルダを選択してください` を初期値にしたプルダウンを置く
- プルダウンには作成済みフォルダを表示し、選択された1フォルダだけ操作UIを表示する
- 操作ボタンの並びは以下とする
  - `招待URL発行（7日）`
  - `招待URL失効`
  - リンク欄内: `コピー`
  - `フォルダパスワード（空で解除）` 入力
  - `設定/解除`
  - `削除`
- 各フォルダ行に `メンバー` 表示を追加する
- フォルダ招待URLで参加したメンバーは、対象フォルダのみ閲覧できる
- フォルダ招待メンバーには、閲覧範囲変更セレクトを表示しない
- フォルダパスワード設定/解除は、削除操作の直上に表示する
- フォルダパスワードは空で更新した場合に解除する
- 招待URL発行時は従来どおり自動コピーし、発行後もリンク欄内の `コピー` から再コピーできる
- 選択フォルダのメンバー表示では、フォルダ招待メンバーだけに `このフォルダから外す` ボタンを表示する
- 管理者とお部屋メンバーはフォルダ管理では削除しない
- フォルダ招待メンバーを対象フォルダから外した後、閲覧可能なフォルダが残っていなければ `status:left` にする

`パスワード設定` の確認文言:

`フォルダ「{フォルダ名}」のパスワードを設定してよかですか？`

`パスワード解除` の確認文言:

`フォルダ「{フォルダ名}」のパスワードを解除してよかですか？`

権限:

- 管理者: 全フォルダで `パスワード設定/解除` / `削除` を表示・実行できる
- メンバー: `パスワード設定/解除` / `削除` は表示しない

補足:

- 今回はメンバーに「自分が作成したフォルダだけ解除可能」は採用しない
- 既存のフォルダ削除権限に合わせ、管理画面のフォルダ操作は管理者専用で揃える

### 5-4. お部屋管理カード構造

`team-admin` は単一カードではなく、管理エリアのコンテナとする。
配下に以下の兄弟カードを配置し、カードの入れ子にはしない。

- `お部屋` カード
- `フォルダ` カード
- `容量` カード

`お部屋` カードの表示順:

1. お部屋招待URL発行
2. 発行中のみ `招待URL失効`
3. 招待URLリンク欄内の `コピー`
4. お部屋メンバー一覧
5. お部屋削除

お部屋メンバー一覧:

- `left` のメンバーは削除済みとして表示しない
- `削除` は物理削除ではなく `PUT /team/members/{userKey}` の `status:left` 更新を再実行する
- 管理者と部屋作成者には `削除` ボタンを表示しない
- フォルダ招待メンバーはお部屋メンバー一覧の削除対象にしない

`フォルダ` カードの表示順:

1. 対象フォルダ選択プルダウン
2. 選択フォルダのブロック
3. フォルダ招待URL発行
4. 発行中のみ `招待URL失効`
5. 招待URLリンク欄内の `コピー`
6. フォルダ招待メンバー一覧（フォルダ招待メンバーだけ解除可能）
7. `フォルダパスワード（空で解除）` と `設定/解除`
8. フォルダ削除

管理操作の完了後はダイアログで完了を知らせ、対象一覧を再取得する。

- お部屋メンバー削除: `メンバーを削除しました。`
- フォルダメンバー解除: `フォルダメンバーを外しました。`
- 閲覧権限変更: 変更前に確認し、完了後 `閲覧権限を更新しました。`
- フォルダ削除: `フォルダを削除しました。`
- フォルダパスワード設定: `フォルダのパスワードを設定しました。`
- フォルダパスワード解除: `フォルダのパスワードを解除しました。`

`容量` カードには、既存の使用量表示、容量グラフ、プラン変更操作、注意書きを移す。
