# 料金プラン差別化 要件整理

更新日: 2026-04-04

## 目的

- フリープランと有料プランの機能差を明確にし、収益性を高める
- 無料でも試せる状態は維持しつつ、継続運用や正式利用では有料プランを選ぶ理由を作る
- 容量だけでなく、フォルダ数、保存期間、PPT出力品質でも差別化する

## 対象プラン

- フリープラン: `512MB`
- 有料プラン: `1GB`
- 有料プラン: `5GB`
- 有料プラン: `10GB`

## 差別化方針

- 無料プランは `お試し利用向け` と位置づける
- 有料プランは `実運用向け` と位置づける
- 制限対象は以下の3項目とする
- フォルダ数
- 保存期間
- PowerPoint 出力時の透かし有無

## プラン別要件

### フリープラン `512MB`

- 作成できるフォルダ数は `2個まで` とする
- 写真データの表示期間は `30日` とする
- `30日` を超えたデータは削除ではなく `アーカイブ` 扱いにする
- アーカイブ済みデータは通常画面では `表示しない`
- PowerPoint 出力時は、写真に `透かし` を入れる

### 有料プラン `1GB / 5GB / 10GB`

- 作成できるフォルダ数は `無制限` とする
- 写真データは `3年保存` とする
- PowerPoint 出力時は、写真に `透かしを入れない`

## 機能要件

### 1. フォルダ数制御

- フリープラン中にフォルダが `2個` ある場合、新規フォルダ作成を不可にする
- 有料プランではフォルダ作成数の上限を設けない
- プラン変更時は、変更後プランの制限を即時反映できること
- 有料プランからフリープランへ戻す場合、既存フォルダ数が `2個以下` の時だけ実行できる
- 既存フォルダ数が `3個以上` の場合は、フリープランへ戻す操作を実行できない
- フリープランへ戻したいユーザーには、`フォルダを2個以下にしてください` と分かる案内を表示する

### 2. 保存期間制御

- フリープランの写真データは、`アップロード日` を基準に `30日` を超えたらアーカイブ対象にする
- アーカイブ対象データは物理削除せず、通常の一覧・表示対象から除外する
- 有料プランでは `3年保存` とし、フリープランのような `30日` アーカイブは行わない
- フリープランで期限切れになって非表示化されたデータは、有料プランに変更したら再表示できるようにする

### 3. PowerPoint 出力制御

- フリープランの PowerPoint 出力では、各写真に透かしを入れる
- 有料プランの PowerPoint 出力では、透かしを入れない
- 透かし画像は [frontend/favicon.png](/Users/aokikensaku/Documents/Devapps/kansa/frontend/favicon.png) を利用する
- 透かしは PowerPoint の各ページ内で、写真の上に並べて表示する
- 透かしは最前面に配置する
- 透かしの透明度は `70%` とする

## 画面・表示要件

- ユーザーが現在プランごとの制限内容を認識できるようにする
- LP に、`プランを横 / 機能を縦` にした比較表を表示する
- LP の比較表では、フリープランと有料プランの差分がひと目で分かるようにする
- お部屋管理画面では、`フリープラン` と `1GB〜10GBプラン` の2区分で制限内容を表示する
- フリープランでは、フォルダ上限、30日保存、有料化で期限切れデータ復元、PPT透かしありを明示する
- 有料プランでは、無制限フォルダ、3年保存、PPT透かしなしを明示する
- LP とお部屋管理画面の保存期間表記は、どちらも `3年保存` に統一する
- 制限到達時は、理由と有料プラン誘導が分かるメッセージを表示する
- お部屋管理画面では、`billing-graph-top-remaining` の近くにフォルダ上限値も明示する
- お部屋管理画面では、以下の容量注記の近くにプラン差分の説明を追加する
- `※ 本サービスの容量表示は2進単位です（1GB=1,024MB、5GB=5,120MB、10GB=10,240MB）。`
- LP とお部屋管理画面の注記では、フリープランへ戻す判定容量に `アーカイブ済みデータを含む` ことを明記する
- フリープランへ戻すボタンは、`使用容量が512MB未満` かつ `フォルダ数が2個以下` の時だけ実行できる
- 条件未達のままフリープランへ戻すボタンを押した場合は、不足条件をユーザー向けメッセージで明示する
- メッセージでは、容量条件とフォルダ条件のどちらが未達かを個別に分かるようにする
- フリープランへ戻す条件チェックは、ボタン押下時に行う

## 運用要件

- 無料プランのアーカイブデータは、有料プランに変更したら再表示する
- アーカイブは `非表示化` と `保持` を両立する設計が望ましい
- 有料プランから無料プランへは、自動解約や自動切り替えではなく、管理画面の `フリープランに戻る` 操作で切り替える
- 有料プランからフリープランへ戻す時は、`使用容量が512MB未満` かつ `フォルダ数が2個以下` の両方を満たす必要がある
- フリープランへ戻す判定に使う `使用容量` は、アーカイブ済みデータを含めて計算する
- 条件を満たさない場合は、フリープランへの変更を完了させない
- 条件未達時は、何を解消すればフリープランへ戻せるかを具体的に案内する
- 管理画面で `フリープランに戻る` を押した時に条件チェックを実行する
- 条件を満たす場合は、フリープランへ切り替えたうえで `戻りました` のダイアログを表示する
- 条件を満たさない場合は、切り替えずに条件未達ダイアログを表示する

## 要確認事項

- なし

## 共通文言方針

- LP 比較表とお部屋管理画面では、保存期間の表現を `3年保存` に統一する
- LP 比較表とお部屋管理画面では、無料プランの表現を `フォルダ2個 / 30日保存 / PPT透かしあり` に統一する
- LP 比較表とお部屋管理画面では、有料プランの表現を `フォルダ無制限 / 3年保存 / PPT透かしなし` に統一する
- LP とお部屋管理画面の注記では、`フリープランへ戻す際の容量判定にはアーカイブ済みデータも含みます` と明記する

## 条件未達時メッセージ要件

- フリープランへの切り替え条件を満たしていない場合は、以下の固定文言を表示する

`フリープランへの切り替えは、以下を満たす必要があります。`

`・容量が512MB未満`

`・フォルダの数が2つ以下`

- 上記に続けて、満たしていない条件だけを表示する
- 容量条件を満たしていない場合は、現在容量が条件を超えていることを分かるように表示する
- フォルダ条件を満たしていない場合は、現在フォルダ数が条件を超えていることを分かるように表示する
- 容量条件とフォルダ条件の両方が未達なら、両方表示する

## フリープラン切り替え成功ダイアログ要件

- 条件を満たしてフリープランへの切り替えが完了した場合は、以下の文言を表示する

`フリープランに戻りました。`

- 必要に応じて、補足として以下の文言を表示する

`現在の上限は、容量512MB未満・フォルダ2個までです。`

## 現時点の整理まとめ

- 無料 `512MB` は `フォルダ2個 / 保存30日 / PPT透かしあり`
- 有料 `1GB・5GB・10GB` は `フォルダ無制限 / 3年保存 / PPT透かしなし`
- 有料から無料へ戻すには `使用容量512MB未満` かつ `フォルダ2個以下` が必須
- フリープランの期限切れデータは非表示にし、有料化で再表示する
- まずはこの差別化を前提に、次段で画面仕様、API制御、保存・アーカイブ仕様へ分解する
