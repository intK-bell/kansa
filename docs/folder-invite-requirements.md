# フォルダ招待URL・メンバー要件

作成日: 2026-04-29

## 1. 背景

現状はお部屋単位で招待URLを発行し、参加したメンバーをお部屋管理から確認・管理できる。
追加要件として、フォルダ単位でも招待URLを発行し、招待されたユーザーが対象フォルダだけを閲覧・利用できるようにする。

## 2. 目的

- お部屋管理メニューのフォルダ一覧に、フォルダごとの `招待URL発行` と `メンバー` を追加する
- フォルダ招待URLから参加したメンバーは、招待されたフォルダだけを閲覧できる
- フォルダ招待メンバーには、閲覧制限の追加設定UIを出さない

## 3. UI 要件

### 3-1. お部屋管理 > フォルダ

各フォルダ行に以下を追加する。

- `招待URL発行（7日）`
- `招待URL失効`
- 招待URL表示欄
- `メンバー` 表示

既存の `パスワード解除` と `削除` は管理者向け操作として維持する。

### 3-2. フォルダメンバー表示

- フォルダごとに、そのフォルダへアクセスできるメンバーを表示する
- 表示対象は以下
  - お部屋管理者
  - 全フォルダ閲覧権限を持つメンバー
  - 自分のフォルダのみ閲覧権限で、そのフォルダを作成したメンバー
  - フォルダ招待URLで対象フォルダへ参加したメンバー
- フォルダ招待メンバーには、閲覧範囲変更セレクトを表示しない

## 4. API 要件

### 4-1. 招待作成

既存の `POST /invites/create` を拡張し、body に `folderId` を渡せるようにする。

- `folderId` なし: 既存どおりお部屋招待
- `folderId` あり: フォルダ招待
- フォルダ招待は、対象フォルダが現在のお部屋に属する場合だけ作成できる
- 招待URLの有効期限は既存どおり7日

### 4-2. 招待失効

既存の `POST /invites/revoke` を利用する。

- token が現在のお部屋の招待であることを確認する
- フォルダ招待の場合も同じ token 失効処理を使う

### 4-3. 招待受諾

既存の `POST /invites/accept` を拡張する。

- お部屋招待: 既存どおり、参加メンバーは全フォルダを閲覧できる
- フォルダ招待: 参加メンバーは対象フォルダだけを閲覧できる
- 同じユーザーが複数のフォルダ招待を受けた場合は、閲覧可能フォルダを追加する
- 管理者はフォルダ招待を受けても閲覧範囲を制限しない

### 4-4. フォルダメンバー一覧

`GET /folders/{folderId}/members` を追加する。

- 管理者のみ実行できる
- 対象フォルダが現在のお部屋に属する場合だけ取得できる
- 返却内容はユーザーキー、表示名、ロール、ステータス、アクセス理由を含む

## 5. 認可要件

- `folderScope = all`: 全フォルダ閲覧可
- `folderScope = own`: 自分が作成したフォルダのみ閲覧可
- `folderScope = invited`: `folderIds` に含まれるフォルダのみ閲覧可
- 管理者は常に全フォルダ閲覧可
- フォルダ招待メンバーは、招待されたフォルダ以外の一覧・写真・アップロード・PPT出力へアクセスできない

## 6. 受け入れ条件

- 管理者がフォルダごとに招待URLを発行できる
- フォルダ招待URLから参加したユーザーのフォルダ一覧には、招待対象フォルダだけが表示される
- 対象外フォルダの写真一覧、アップロード、PPT出力は `403` になる
- フォルダ一覧のメンバー表示に、フォルダ招待で参加したユーザーが表示される
- 部屋招待URLの既存動作は変わらない
