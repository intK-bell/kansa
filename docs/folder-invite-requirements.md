# フォルダ招待URL・メンバー要件

作成日: 2026-04-29

## 1. 背景

現状はお部屋単位で招待URLを発行し、参加したメンバーをお部屋管理から確認・管理できる。
追加要件として、フォルダ単位でも招待URLを発行し、招待されたユーザーが対象フォルダだけを閲覧・利用できるようにする。

## 2. 目的

- お部屋管理メニューのフォルダ一覧に、フォルダごとの `招待URL発行` と `メンバー` を追加する
- フォルダ招待URLから参加したメンバーは、招待されたフォルダだけを閲覧できる
- フォルダ招待メンバーには、閲覧制限の追加設定UIを出さない

## 3. UI 要件

### 3-0. メンバー種別と削除場所

- 1つのお部屋内では、ユーザーの立場は `管理者` / `お部屋メンバー` / `フォルダメンバー` のどれか1つだけとする
- `管理者`: フォルダ単位では削除しない
- `お部屋メンバー`: お部屋管理でだけ削除する。フォルダ管理では削除しない
- `フォルダメンバー`: 対象フォルダの管理でだけ外せる
- フォルダメンバーを対象フォルダから外した後、閲覧可能なフォルダが残っていなければ `status=left` にする
- 既にお部屋メンバーのユーザーがフォルダ招待を受けた場合は、お部屋メンバーのまま扱う
- フォルダメンバーが お部屋招待を受けた場合は、お部屋メンバーへ昇格する

### 3-1. お部屋管理 > フォルダ

各フォルダ行に以下を追加する。

- `招待URL発行（7日）`
- `招待URL失効`
- 招待URL表示欄
- `メンバー` 表示

既存の `パスワード解除` と `削除` は管理者向け操作として維持する。

### 3-2. フォルダメンバー表示

- フォルダごとに、フォルダ招待URLで対象フォルダへ参加したフォルダメンバーだけを表示する
- 管理者とお部屋メンバーは、フォルダメンバー一覧には表示しない
- フォルダ招待メンバーには、閲覧範囲変更セレクトを表示しない
- 削除/解除操作はフォルダ招待メンバーだけに表示する

## 4. API 要件

### 4-1. 招待作成

既存の `POST /invites/create` を拡張し、body に `folderId` を渡せるようにする。

- `folderId` なし: 既存どおりお部屋招待
- `folderId` あり: フォルダ招待
- フォルダ招待は、対象フォルダが現在のお部屋に属する場合だけ作成できる
- 招待URLの有効期限は既存どおり7日

### 4-2. 招待失効

既存の `POST /invites/revoke` を利用する。

- token が現在のお部屋の招待であることを確認する
- フォルダ招待の場合も同じ token 失効処理を使う

### 4-3. 招待受諾

既存の `POST /invites/accept` を拡張する。

- 招待を受ける主体は常に同じ Cognitoユーザーで、永続キーは Cognito `sub` とする
- メールアドレスは表示・連絡用であり、メンバー識別やDB主キーには使わない
- お部屋招待: 既存どおり、参加メンバーは全フォルダを閲覧できる
- フォルダ招待: 参加メンバーは対象フォルダだけを閲覧できる
- 同じユーザーが複数のフォルダ招待を受けた場合は、閲覧可能フォルダを追加する
- 管理者はフォルダ招待を受けても閲覧範囲を制限しない
- 既に `folderScope=all` のお部屋メンバーがフォルダ招待を受けた場合は、権限を変更しない
- 既に `folderScope=own` のお部屋メンバーがフォルダ招待を受けた場合は、対象フォルダを `folderIds` に追加し、お部屋メンバーのまま閲覧可能にする
- `folderScope=invited` のフォルダメンバーが お部屋招待を受けた場合は、`folderScope=all` に変更し、`folderIds` を空にする
- `status=left` のユーザーが再招待された場合は、物理的に作り直さず `active` に戻して再参加させる

### 4-4. フォルダメンバー一覧

`GET /folders/{folderId}/members` と `DELETE /folders/{folderId}/members/{userKey}` を追加する。

- 管理者のみ実行できる
- 対象フォルダが現在のお部屋に属する場合だけ取得できる
- 返却内容はユーザーキー、表示名、ロール、ステータス、アクセス理由を含む
- フォルダメンバー解除APIは、`folderScope=invited` のメンバーだけを対象にする
- 解除時は対象 `folderId` を `folderIds` から外し、残りが空なら `status=left` にする

## 5. 認可要件

- `folderScope = all`: 全フォルダ閲覧可
- `folderScope = own`: 自分が作成したフォルダのみ閲覧可
- `folderScope = invited`: `folderIds` に含まれるフォルダのみ閲覧可
- 管理者は常に全フォルダ閲覧可
- フォルダ招待メンバーは、招待されたフォルダ以外の一覧・写真・アップロード・PPT出力へアクセスできない

## 6. 受け入れ条件

- 管理者がフォルダごとに招待URLを発行できる
- フォルダ招待URLから参加したユーザーのフォルダ一覧には、招待対象フォルダだけが表示される
- 対象外フォルダの写真一覧、アップロード、PPT出力は `403` になる
- フォルダ一覧のメンバー表示に、フォルダ招待で参加したユーザーが表示される
- 部屋招待URLの既存動作は変わらない
